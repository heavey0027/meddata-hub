## 后端接口列表

| 文件名       | 接口路径                                  | 操作方式 | 描述                                                         |
| :----------- | :---------------------------------------- | :------: | :----------------------------------------------------------- |
| `patient.py` | `/api/patients`                           |  `GET`   | 获取所有患者信息，支持通过 `query` 参数查询特定患者ID，以及通过 `limit` 和 `offset` 参数进行分页。返回的患者信息包含 `is_vip` 字段，表示该患者是否去过所有科室。 |
| `patient.py` | `/api/patients`                           |  `POST`  | 新增/注册患者。                                              |
| `patient.py` | `/api/patients/<string:p_id>`             |  `PUT`   | 更新指定患者ID (`p_id`) 的信息。。                           |
| `patient.py` | `/api/patients/<string:patient_id>`       | `DELETE` | 删除指定患者ID (`patient_id`) 及其所有关联数据，包括挂号记录和病历记录（病历记录的删除会级联删除处方明细）。 |
| `record.py`  | `/api/records`                            |  `GET`   | 获取所有或某个患者的病历记录。                               |
| `record.py`  | `/api/prescription_details`               |  `GET`   | 获取所有或某个病历的处方细则。                               |
| `record.py`  | `/api/records`                            |  `POST`  | 提交新的病历及关联的处方细则。                               |
| `record.py`  | `/api/records/<string:record_id>`         | `DELETE` | 删除指定病历ID (`record_id`)。此操作会级联删除该病历下的所有处方明细。 |
| `basic.py`   | `/api/departments`                        |  `GET`   | 获取所有科室信息，包含 id, name, location。                  |
| `basic.py`   | `/api/departments/<string:department_id>` | `DELETE` | 删除指定科室ID (`department_id`)。在删除前会检查该科室是否有医生，如果存在医生则不允许删除。 |
| `basic.py`   | `/api/medicines`                          |  `GET`   | 获取所有所有药品信息，包含 id, name, price, stock, specification。 |
| `basic.py`   | `/api/medicines/<string:medicine_id>`     | `DELETE` | 删除指定药品ID (`medicine_id`)。在删除前会检查该药品是否有任何相关的处方细则，如果存在则不允许删除。 |
| `doctor.py`  | `/api/doctors`                            |  `GET`   | 获取所有医生信息。                                           |
| `doctor.py`  | `/api/doctors/<string:doctor_id>`         | `DELETE` | 删除指定医生ID (`doctor_id`)。在删除前会检查该医生是否有任何相关的挂号记录或病历记录，如果存在则不允许删除。 |
